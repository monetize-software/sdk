# Контекст проекта

Полная документация проекта — в [README.md](README.md): описание, стек, команды, структура `platform`/`online`/`docs`, БД, деплой, кастомные домены. Также залита в Cognee KG (датасет `monetize_software`). Читай README.md когда нужен контекст по стеку/структуре/командам, граф — для смысловых запросов «как у нас это устроено» и «что мы решили по X».

# Правила разработки

- Path alias: `@/*` → корень проекта (в обоих проектах)
- `strict: false` в tsconfig — не включать strict
- Форматирование: single quotes, 2 пробела, без trailing commas (Prettier)
- online должен оставаться максимально лёгким — минимум зависимостей, быстрая загрузка
- В online для клиентского хранилища использовать `clientStorage` из `@/utils/clientStorage` вместо прямого `localStorage`. Ключи в `STORAGE_KEYS` в том же файле.
- Язык коммуникации: русский

# Автозапись в Cognee KG

Когда в обсуждении или работе ловишь **ценную информацию, которой нет в файлах** — фиксируй её в граф **без отдельной просьбы пользователя**. Контекст без записи теряется в следующей сессии.

**Что считать ценным:**
- Архитектурное решение и **причина** выбора («Preact вместо React — критично для bundle»; «не уходим с Supabase сейчас, потому что 59+ RLS политик»).
- Корнер-кейс или баг, замеченный в обсуждении и которого нет в коде/комментариях.
- Договорённость или приоритет, влияющий на будущие задачи.
- Открытие про внешнюю систему (Stripe, Supabase, Cloudflare, CWS), полезное для последующих работ.
- Нетривиальное «почему так», которое не очевидно из кода.

**Что НЕ записывать:**
- Уже покрыто `README.md`, `MEMORY.md`, `TODO.md` или самим кодом.
- Ход выполненной задачи (это в git/PR/TODO.md «Сделано»).
- Дубли — перед записью прогоняй `mcp__cognee__recall` похожего запроса и проверяй.

**Как записывать (паттерн):**
- Содержательное обсуждение → `mcp__cognee__save_interaction` с компактным «Q: ... / A: ...».
- Новый файл/документ → `mcp__cognee__remember(content c заголовком "Source: <абс. путь>", dataset_name='monetize_software')`.
- Изменился ранее залитый файл → `list_data` → `delete(data_id, mode='soft')` → `remember` новой версии (параллельные `remember` в один датасет ловят UNIQUE-constraint — делать последовательно).
- Подробности и search modes — в `~/.claude/projects/-Users-aleksandr-WebstormProjects-monetize-software/memory/reference_cognee_kg.md`.

**Дисциплина:** не превращай граф в помойку. Если запись пограничная — спроси «есть смысл это записать в cognee?» прежде чем фиксировать. После записи — кратко скажи пользователю, что и почему сохранил.
